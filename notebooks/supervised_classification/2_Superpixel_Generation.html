So you may ask what the transpose was for. Well, gdal and numpy require arrays of slightly different format. The former reads arrays as Band Interleaved by Pixel (BIP) and the latter, Band Sequential (BSQ) (see here). The landsat has 4 bands (or channels). Rasterio (a gdal wrapper) reads the array with the channels at the front, but numpy requires these channels to be at the back.

The superpixel segmentation takes several parameters that can be read about in the docs. To summarize, the min_size specifies the minimum size of all segments in the image, scale controls the approximate irregularity of the segments (precisely, a larger scale parameter determines if smaller sized segments are merged more easily during the formation of the final segmentation), and sigma is set to 0 so there is no smoothing done before segmentation.
